## Commands

```bash
# Development
npm run dev              # Start Next.js dev server (port 3000)
npm run dev:compose      # Start via Docker Compose (app + Swagger UI on port 8080)

# Build & Production
npm run build
npm start

# Testing
npm run test             # Jest (single run)
npm run test:watch       # Jest watch mode
npm run test:coverage    # Jest with coverage report
npm run playwright:test  # Playwright E2E tests
npm run playwright:test:ui  # Playwright with UI

# Code Quality
npm run lint

# Code Generation
npm run orval            # Regenerate API hooks/types from OpenAPI spec
npm run generate:swagger # Generate TypeScript client via Docker

# Storybook
npm run storybook        # Start on port 6006
```

To run a single Jest test file:
```bash
npx jest src/path/to/file.test.tsx
```

## Architecture

This is a **Next.js 15 App Router** application with API routes acting as a backend — there is no separate server process.

### Key technology choices

- **State:** Jotai atoms (`src/app/atoms/`)
- **Data fetching:** TanStack React Query, with hooks auto-generated by Orval from `src/app/openApi/openapiSample.yml` → `src/app/openApi/generated/`
- **Forms:** React Hook Form + Yup validation
- **UI:** PrimeReact components + Primeflex CSS grid + Styled Components
- **Auth:** NextAuth (`/api/auth/[...nextauth]/`)
- **HTTP client:** custom Axios instance at `lib/axios.ts` (also used as the Orval mutator)

### Component structure (Atomic Design)

```
src/app/components/
├── atoms/      # Single-purpose UI components; each has .tsx + .test.tsx + .stories.tsx
├── molecules/  # Combinations of atoms
├── organisms/  # Complex sections (e.g. NavigationMegaMenu)
├── auth/
└── provider/   # React context providers
```

### Route layout

- `src/app/(page)/` — Frontend page routes
- `src/app/orders/[orderId]/` — Dynamic order route
- `src/app/api/` — API route handlers (affiliations, employees, export/csv, upload/csv, orders, stats, register, zod-sample)
- `src/app/layout.tsx` — Root layout; wraps everything in providers, mounts Toast and NavigationMegaMenu

### Testing setup

- Jest uses `@happy-dom/jest-environment` (not jsdom — avoids conflicts with Playwright)
- Playwright looks for tests inside `src/app/(page)/`
- Setup file: `jest.setup.ts` (imports `@testing-library/jest-dom`)

### Path alias

`@/*` resolves to `src/*` (configured in `tsconfig.json` and `jest.config.js`).

### OpenAPI workflow

Edit `src/app/openApi/openapiSample.yml` → run `npm run orval` → generated React Query hooks and Zod schemas appear in `src/app/openApi/generated/`. Do not hand-edit generated files.
