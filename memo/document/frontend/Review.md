# SPA（Next.js）製造レビュー チェックリスト

実装レビューで使える観点を体系化しています。コピペしてそのまま利用可能。

---

## 1. 型安全

| 観点                                  | チェック |
| ------------------------------------- | -------- |
| anyを使っていない                     | ☐        |
| asによる型アサーションを避けている    | ☐        |
| union型を使っている                   | ☐        |
| enumや定数化している                  | ☐        |
| 型ガードを利用している                | ☐        |
| satisfiesを使える箇所で使っている     | ☐        |
| 外部APIの型が安全に定義されている     | ☐        |
| nullとundefinedの扱いが統一されている | ☐        |

---

## 2. JavaScript / TypeScript 基本

| 観点                                      | チェック |
| ----------------------------------------- | -------- |
| Optional Chaining（?.）を適切に使用       | ☐        |
| Nullish Coalescing（??）を使用            | ☐        |
| === を使用                                | ☐        |
| 空文字・undefinedの扱いが明確             | ☐        |
| マジックナンバーを使用していない          | ☐        |
| Promiseにawaitをつけ忘れていない          | ☐        |
| console.logが残っていない                 | ☐        |
| 破壊的変更（push / sortなど）を避けている | ☐        |

---

## 3. React 基本

| 観点                                   | チェック |
| -------------------------------------- | -------- |
| keyを忘れていない                      | ☐        |
| keyにindexを使用していない             | ☐        |
| 再レンダリング事故を防いでいる         | ☐        |
| オブジェクト・関数の直書きを避けている | ☐        |
| 必要に応じてmemo / useMemoを使用       | ☐        |
| useCallbackが適切に使われている        | ☐        |

---

## 4. 副作用管理

| 観点                             | チェック |
| -------------------------------- | -------- |
| 副作用と純粋関数が分離されている | ☐        |
| 不要なuseEffectを使っていない    | ☐        |
| useEffectの依存配列が正しい      | ☐        |
| API呼び出しがhooks化されている   | ☐        |
| 非同期エラーを握り潰していない   | ☐        |

---

## 5. 状態管理

| 観点                                   | チェック |
| -------------------------------------- | -------- |
| 状態の正規化がされている               | ☐        |
| derived stateを持っていない            | ☐        |
| UI状態とサーバーデータが分離されている | ☐        |
| グローバルstateを乱用していない        | ☐        |
| キャッシュとstateが混ざっていない      | ☐        |

---

## 6. パフォーマンス

| ローディング状態が適切             | ☐        |
